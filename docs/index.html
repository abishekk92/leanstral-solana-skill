How it works

Claude drives proof writing directly. No lossy analyzer, no pipeline, no prompt templates.

1

Claude reads your code

Claude reads your Solana source, writes a verification spec (SPEC.md), and models your program's state, transitions, and invariants in Lean 4.

2

Proofs are written and compiled

Claude writes theorem statements and proofs, then runs lake build and iterates on errors. Full code context means better proofs on the first try.

3

Hard sub-goals go to Leanstral

When Claude can't fill a sorry, it calls Leanstral (Mistral's theorem prover) via fill-sorry with pass@N sampling. First compilable result wins.

What gets verified

Six categories of properties, proven as Lean 4 theorems that must compile.

Access Control

Signer checks and authority constraints. Only authorized parties can call each instruction.

Conservation

Token totals preserved across every operation. The vault always covers outstanding claims.

CPI Correctness

Every cross-program invocation passes the correct program ID, accounts, authority, and amount.

State Machines

Lifecycle transitions are deterministic. Closed accounts stay closed, modes follow prescribed order.

Arithmetic Safety

No silent overflow or underflow. All u64/i128 operations proven to stay within bounds.

Fee Isolation

Unpaid fees tracked as negative credits, never inflating PnL, insurance, or deficit calculations.

We verify program logic. SPL Token, the Solana runtime, CPI mechanics, and Anchor are trusted as axioms.

Escrow

Anchor

A token escrow that lets two parties trade safely. The initializer deposits tokens and sets terms, a taker completes the trade, or the initializer cancels.

-- Cancel only succeeds if signer = initializer
theorem cancel_access_control
    (p_preState : ProgramState) (p_signer : Pubkey)
    (h : cancelTransition p_preState p_signer ≠ none) :
    p_signer = p_preState.escrow.initializer := by
  unfold cancelTransition at h
  split_ifs at h with h_eq
  · exact h_eq
  · contradiction

-- Deposit preserves V >= C_tot + I
theorem deposit_conservation
    (p_s p_s' : EngineState) (p_amount : Nat)
    (h_inv : conservation p_s)
    (h : depositTransition p_s p_amount = some p_s') :
    conservation p_s' := by
  unfold depositTransition at h
  split_ifs at h with h_le
  cases h
  unfold conservation at h_inv ⊢
  dsimp only
  omega

Get started

Three steps to formally verified Solana programs.

1. Set your Mistral API key

Free during Mistral's feedback period.

export MISTRAL_API_KEY=your_key_here

2. Build the CLI

git clone https://github.com/abishekk92/leanstral-solana-skill.git
cd leanstral-solana-skill
cargo build --release

3. Ask Claude to verify your program

Open Claude Code in your Solana project directory and say:

Formally verify this program.

Claude will read your code, write SPEC.md, generate Lean 4 proofs, and iterate until they compile.

First Lean build takes 15-45 minutes to compile Mathlib. Subsequent builds reuse the cache.
